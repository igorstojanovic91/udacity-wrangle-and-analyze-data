\section{Wrangle Report}\label{wrangle-report}}

    This wrangle report takes aim to provide report about the data wrangling
that was conducted within the Data Analyst Nanodegree Term 2 Project
``Wrangle and Analyze Data''.

    After first gathering data from different sources, the author found
himself in front of 3 datasets, which were unclean and untidy. In order
to proceed to future analyses, conducting good data wrangling was
crucial.

    After assessing the data set, the author outlined the following
shortcomings:

\hypertarget{summary-assessing}{%
\subsection{Summary Assessing}\label{summary-assessing}}

\hypertarget{visual-assesment}{%
\subsubsection{Visual assesment}\label{visual-assesment}}

\textbf{Quality:}

\begin{itemize}
\tightlist
\item
  Some tweet\_ids contain links instead of numbers
\item
  Some sources contain multiple links and are hardly readable
\item
  Some Text is missing
\item
  Retweeted columns seem to miss a lot of values
\item
  Some expanded URLs have multiple entries (the same link was repeated
  several times)
\item
  Some rating numerators and denominators are missing
\item
  Some names are missing
\item
  Little information about doggo, floofer, pupper, puppo is available
\item
  Missing information of twitter\_id\_enhanced is not part of the data
  set
\item
  There were no issues identifliied in the image data set
\end{itemize}

\hypertarget{programmatic-asssesment}{%
\subsubsection{Programmatic Asssesment}\label{programmatic-asssesment}}

\textbf{Quality:}

\begin{itemize}
\tightlist
\item
  \texttt{Tweets\_archive} show more rows than \texttt{tweets\_data} (15
  in total), which is the exact amount of exeptions
\item
  \texttt{Tweet\_ip} has significant lower amount of rows: 2075 in total
\item
  Ids in exeption not part of \texttt{tweets\_data}
\item
  \texttt{tweets\_data} id should be string and not an integer
\item
  \texttt{tweets\_data} created\_at should be pd.date\_time format
  instead of string
\item
  \texttt{twitter\_archive} tweet\_id should be string and not an
  integer
\item
  Texts in both datasets \texttt{twitter\_archive} and
  \texttt{tweets\_data} are not in full length
\item
  There is a really high numerator and denumerator in
  \texttt{twitter\_archive}
\item
  Exanded\_urls in \texttt{twitter\_archive} do not work
\item
  \texttt{twitter\_archive} in\_reply\_to\_status\_id and
  in\_reply\_to\_user\_id contain only 78 non-null values
\item
  \texttt{twitter\_archive} contain only 181 non-null values of
  retweeted\_status\_id, retweeted\_status\_user\_id and
  retweeted\_status\_timestamp
\end{itemize}

\textbf{tidiness}

\begin{itemize}
\tightlist
\item
  One single column for``stages od dog'' (doggo floofer pupper puppo) in
  \texttt{twitter\_archive}
\item
  All files should be joined to one ``Master file''
\end{itemize}

    One can imagine by reading trough all this shortcomings that conducting
analysis would be rather a pain than a join. So the author tackled the
problem step by step:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  The correct urls were extracted from the twitter data set. This allows
  reassurance in case the numbers seem not to be right.
\item
  The quality of stage\_of\_dog was improve by extracting the
  information abount the stage from the twitter text
\item
  Stage\_of\_dog was set to one variable instead of former 4
\item
  Since there was missing and probably wrong information in the archive,
  the nominator and denominator were again extracted from the text.
\item
  Unwanted columns of the data were removed
\item
  Twitter data coloumns were classified accordingly (string, datetime)
\item
  All datasets were merged
\item
  Differences in numerator and denuminator column were identified and
  cleaned
\item
  Source column was made more readable
\end{enumerate}

    The application of the learned data wrangling skills made this possible
and prepared the data for a quick analysis.
